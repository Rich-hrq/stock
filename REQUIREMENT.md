# 基金持仓记录功能 — 需求档案

## 1. 功能概述

在现有美股指数分析网站基础上，新增**基金持仓记录**功能。用户可以注册登录后，记录自己在4个美股指数上的买入/卖出操作，系统自动查询当日收盘价和汇率，计算持仓份额、已实现盈亏和假设当日清仓的预期盈亏。

## 2. 用户系统

### 2.1 注册
- 用户名（唯一，3-20字符）
- 密码（6位以上，bcrypt 加密存储）

### 2.2 登录
- 用户名 + 密码登录
- 登录后通过 JWT Token 鉴权（存储在 localStorage）
- Token 过期时间：7天

### 2.3 权限隔离
- 未登录用户：可正常使用原有的指数分析、RAG问答、预测市场、新闻等功能（不受影响）
- 已登录用户：额外可访问持仓记录功能
- 每个用户只能看到自己的交易数据

## 3. 交易记录

### 3.1 支持的标的
- 标普500（^GSPC）
- 纳斯达克100（^NDX）
- 纳斯达克综合指数（^IXIC）
- 道琼斯工业指数（^DJI）

### 3.2 每笔交易字段

| 字段 | 说明 | 来源 |
|------|------|------|
| 指数 | 上述4个指数之一 | 用户选择（下拉框） |
| 方向 | 买入 / 卖出 | 用户选择 |
| 日期 | 交易日期 | 用户选择（日期选择器） |
| 人民币金额 | 本次买入/卖出的人民币金额 | 用户输入 |
| 当日收盘价 | 该指数当日的收盘价（美元） | 系统自动查询 yfinance |
| 当日汇率 | USD/CNY 汇率 | 系统自动查询 |
| 美元等值 | 人民币金额 ÷ 汇率 | 系统自动计算 |
| 持有份额 | 美元等值 ÷ 当日收盘价 | 系统自动计算 |
| 创建时间 | 记录创建时间 | 系统自动 |

### 3.3 汇率数据来源
- API：`https://open.er-api.com/v6/latest/USD`（免费、无需 Key、支持 CORS）
- 返回的 `rates.CNY` 即为当前 USD/CNY 汇率
- **历史日期处理**：免费 API 不支持按日期查询历史汇率，因此：
  - 系统默认使用当前实时汇率填入（作为近似值）
  - 用户可在提交前手动修改汇率字段
  - 汇率一旦记录，后续不再自动更新（以记录时的值为准）

### 3.4 收盘价数据来源
- 复用现有 yfinance 服务（`services/market_data.py`）
- 查询交易日期当日的收盘价
- 若当日非交易日，取最近一个交易日的收盘价

## 4. 计算逻辑

### 4.1 持仓份额（加权平均法）

每次买入增加份额，每次卖出减少份额：

```
买入：新增份额 = 人民币金额 ÷ 汇率 ÷ 当日收盘价
     总份额 += 新增份额
     总成本（人民币） += 人民币金额

卖出：卖出份额 = 人民币金额 ÷ 汇率 ÷ 当日收盘价
     总份额 -= 卖出份额
     总成本（人民币） -= (加权平均成本 × 卖出份额)
```

### 4.2 加权平均成本

```
加权平均成本（人民币/份额） = 总成本（人民币） ÷ 总份额
```

### 4.3 已实现净赚

每笔卖出的已实现收益：

```
已实现收益 = 卖出人民币金额 - (加权平均成本 × 卖出份额)
```

累计所有卖出交易的已实现收益之和即为**已实现净赚**。

### 4.4 假设当日清仓净赚

```
当日清仓金额 = 总份额 × 当前收盘价 × 当日汇率
未实现盈亏 = 当日清仓金额 - 当前持仓成本（人民币）
总盈亏 = 已实现净赚 + 未实现盈亏
```

### 5.3 交互说明
- 提交交易后，表单清空，持仓汇总和历史列表实时刷新
- 持仓汇总中，盈利显示绿色，亏损显示红色
- 历史列表支持按时间倒序排列（最新在前）
- 当日清仓的"当前收盘价"实时从后端获取

## 7. 数据库设计

### 7.1 users 表

| 列名 | 类型 | 说明 |
|------|------|------|
| id | INT AUTO_INCREMENT PK | 主键 |
| username | VARCHAR(20) UNIQUE | 用户名 |
| password_hash | VARCHAR(128) | bcrypt 加密密码 |
| created_at | DATETIME | 注册时间 |

### 7.2 transactions 表

| 列名 | 类型 | 说明 |
|------|------|------|
| id | INT AUTO_INCREMENT PK | 主键 |
| user_id | INT FK → users.id | 所属用户 |
| symbol | VARCHAR(20) | 指数代码（如 ^GSPC） |
| direction | ENUM('buy', 'sell') | 买入/卖出 |
| trade_date | DATE | 交易日期 |
| amount_cny | DECIMAL(12,2) | 人民币金额 |
| close_price_usd | DECIMAL(10,2) | 当日收盘价（美元） |
| exchange_rate | DECIMAL(8,4) | 当日 USD/CNY 汇率 |
| usd_equivalent | DECIMAL(12,2) | 美元等值（amount_cny / rate） |
| shares | DECIMAL(12,6) | 持有份额（usd_equivalent / close_price） |
| created_at | DATETIME | 记录创建时间 |

## 8. 技术选型

| 组件 | 方案 |
|------|------|
| 后端框架 | FastAPI（复用现有） |
| 数据库 | MySQL（远程，配置留空） |
| ORM | SQLAlchemy + aiomysql |
| 鉴权 | JWT（python-jose） |
| 密码加密 | bcrypt（passlib） |
| 汇率 API | open.er-api.com（免费、无需 Key） |
| 收盘价 | yfinance（复用现有 market_data.py） |
| 前端 | 纯 HTML/CSS/JS（与现有风格一致） |

## 9. 与现有功能的关系

- **完全独立**：新增的用户系统和持仓功能不影响现有页面
- **按需登录**：只有访问持仓记录功能时才需要登录，其他功能对未登录用户完全开放
- **现有页面不改动**：index.html、prediction.html、news.html 的导航栏保持原样，不加登录按钮

### 9.1 导航与登录流程

现有页面结构（各自独立，不修改）：
- `index.html`：header 内有 "预测市场"、"新闻资讯" 链接
- `prediction.html` / `news.html`：各自有 topbar，带"返回主页"链接

新增页面 `portfolio.html`：
- 有自己的 topbar，带"返回主页"链接
- 页面上显示"登录 / 注册"按钮（topbar 右侧）
- **未登录时**：点击"持仓记录"链接 → 跳转到 portfolio.html → 页面显示登录/注册表单（居中卡片），不显示交易内容
- **登录后**：页面显示完整的交易记录、持仓汇总等功能
- 登录状态保存在 localStorage（JWT Token），其他页面不感知登录状态

### 9.2 从主页进入持仓记录

在 `index.html` 的 header 中新增一个导航链接"持仓记录"（`<a href="/portfolio.html">`），与现有"预测市场"、"新闻资讯"链接并列。用户点击后跳转到 portfolio.html。

## 10. 删除交易

- 允许删除已有的交易记录
- 删除后，系统自动重新计算该用户所有剩余交易的持仓份额和盈亏
- 删除操作需二次确认（弹窗）
